# 🚀 AI社交媒体内容分析系统 部署手册
## 📋 前置准备（需要提前准备好）
1. **云服务器**：配置建议2核4G及以上，操作系统CentOS 7+/Ubuntu 20.04+
2. **服务器公网IP**：比如 `123.123.123.123`
3. **API密钥**：TikHub API密钥、火山引擎豆包API密钥
4. **（可选）域名**：如果需要绑定域名提前解析到服务器IP

---

## 🔧 第一步：安装服务器环境（仅首次部署需要）
### 1. 登录服务器
用SSH工具（FinalShell/Xshell/终端）登录到你的服务器，登录后会看到类似 `[root@xxx ~]#` 的提示符。

### 2. 安装Docker
复制下面的命令粘贴执行：
```bash
curl -fsSL https://get.docker.com | bash
```
等待执行完成，出现 `Installation complete` 说明成功。

### 3. 启动Docker并设置开机自启
```bash
systemctl start docker
systemctl enable docker
```

### 4. 安装Docker Compose
```bash
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 5. 验证安装是否成功
分别执行下面两个命令，都出现版本号说明环境安装完成：
```bash
docker --version
# 示例输出：Docker version 25.0.3, build 4debf41
docker-compose --version
# 示例输出：Docker Compose version v2.24.6
```

## 🚢 第二步：部署系统
### 1. 拉取代码
```bash
cd /opt
git clone https://github.com/HiOne0826/douyin-ai-analyzer.git
cd douyin-ai-analyzer
```

### 2. 配置环境变量（最重要的一步）
```bash
cp backend/.env.example backend/.env
vim backend/.env
```
按 `i` 进入编辑模式，修改下面两个配置为真实的密钥，其他配置保持默认：
```env
TIKHUB_API_KEY=这里填你的TikHub API密钥
ARK_API_KEY=这里填你的豆包API密钥
DEBUG=false
```
修改完成后按 `ESC`，输入 `:wq` 回车保存退出。

### 3. 一键启动服务
```bash
cd deploy
docker-compose up -d
```
第一次启动会自动下载镜像和构建，等待2-3分钟。

## ✅ 第三步：验证部署成功
### 1. 查看服务状态
```bash
docker-compose ps
```
看到三个服务（backend/frontend/nginx）的STATE都是 `Up` 说明启动成功。

### 2. 访问系统
打开浏览器访问你的服务器公网IP：`http://你的服务器IP`，就能看到系统页面。

### 3. 测试功能
随便找一个抖音/小红书链接测试：
1. 输入链接点击「获取内容」，能获取到数据说明后端正常
2. 选择图片点击「AI分析」，能看到流式输出说明所有功能正常

## 🌐 （可选）绑定域名+HTTPS
### 1. 域名解析
把你的域名（比如 `ai.example.com`）解析到服务器IP。

### 2. 修改Nginx配置
```bash
cd /opt/douyin-ai-analyzer/deploy
vim nginx.conf
```
把第二行的 `server_name _;` 改为你的域名：
```nginx
server_name ai.example.com;
```
保存退出。

### 3. 申请SSL证书
```bash
# CentOS系统执行
yum install -y certbot python3-certbot-nginx
# Ubuntu系统执行
apt install -y certbot python3-certbot-nginx

# 申请证书，按照提示操作
certbot --nginx -d ai.example.com
```

### 4. 重启服务生效
```bash
docker-compose restart nginx
```
现在访问 `https://ai.example.com` 就是HTTPS加密访问了。

## 🔧 日常维护命令
所有命令都需要先进入 `deploy` 目录执行：
```bash
cd /opt/douyin-ai-analyzer/deploy
```

| 命令 | 作用 |
|------|------|
| `docker-compose logs -f` | 查看所有服务实时日志 |
| `docker-compose logs -f backend` | 只看后端日志 |
| `docker-compose restart` | 重启所有服务 |
| `docker-compose down` | 停止所有服务 |
| `docker-compose up -d --build` | 更新代码后重新构建启动 |

### 更新系统版本
```bash
cd /opt/douyin-ai-analyzer
git pull
cd deploy
docker-compose up -d --build
```

## ❌ 常见问题排查
1. **访问不了页面**：检查服务器安全组是否开放了80/443端口
2. **获取内容失败**：查看后端日志 `docker-compose logs -f backend`，检查TikHub API密钥是否正确
3. **AI分析失败**：检查豆包API密钥是否正确，是否有调用额度
4. **页面加载慢**：检查服务器网络是否正常，是否能正常访问GitHub/火山引擎API
